# Генератор макетов

Статическое веб-приложение для сборки рекламных баннеров и других макетов в разных форматах. Интерфейс работает без сборщика: `index.html` подключает ES-модули из `src/`, а рендер происходит на canvas.

## Что есть в проекте

- генерация макетов в нескольких форматах и пресетах размеров;
- работа с логотипами, KV, фонами, кастомными шрифтами и цветовыми настройками;
- локальный импорт `config.json` при старте приложения;
- экспорт PNG/JPG;
- регрессионный Playwright-тест для wide-layout сценариев без KV;
- деплой в Yandex Object Storage через `deploy.sh`.

## Требования

- Python 3.x для локального сервера и вспомогательных скриптов;
- современный браузер с поддержкой ES-модулей;
- опционально `Pillow` для конвертации изображений в локальном сервере и deploy-пайплайне;
- опционально Playwright Test для запуска теста из `tests/`.

## Локальный запуск

### Рекомендуемый способ

```bash
python3 start_server.py
```

или

```bash
./start_server.py
```

Скрипт поднимает локальный HTTP-сервер, добавляет CORS-заголовки, умеет обслуживать API-эндпоинты для файловых операций и автоматически открывает приложение в браузере. По умолчанию используется `http://localhost:8000`; если порт занят, скрипт выбирает следующий свободный.

### Упрощённый способ

```bash
python3 -m http.server 8000
```

Этот вариант подходит для базового просмотра интерфейса, но не заменяет `start_server.py`, если нужны серверные операции, которые ожидает UI.

## Конфигурация

- Файл `config.json` в корне проекта необязателен.
- Пример структуры лежит в `config.json.example`.
- При наличии `config.json` приложение читает его на старте через `src/utils/fullConfig.js`.

## Тесты

В проекте есть регрессионный тест:

```bash
npm install
npm run test:regression
```

Конфигурация лежит в `playwright.config.mjs`. Зависимость `playwright` описана в `package.json`. Во время теста поднимается локальный сервер на `http://127.0.0.1:4173`.

## Деплой

Основной сценарий:

```bash
./deploy.sh
```

Для полной синхронизации бакета:

```bash
./deploy.sh --full
```

Подробности вынесены в `DEPLOY.md`, типовые ошибки и восстановление после проблем описаны в `TROUBLESHOOTING.md`.
